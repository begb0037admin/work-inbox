## Architecture

| Component | Description |
|-----------|-------------|
| fetch_inbox.py | Three-phase script. Phase 1: Outlook COM via pywin32, pulls inbox/sent/calendar to data/inbox_raw.json. Phase 2: calls Anthropic API (claude-haiku-4-5) for triage, injects EntryIDs by subject match, writes data/briefing.json. Phase 3: pushes briefing.json to GitHub via Contents API (PAT from GITHUB_PAT env var). No browser open. |
| open_email.py | Protocol handler for openmail:// custom URL scheme. Strips prefix and trailing slash (Chrome appends /), opens exact email in classic Outlook via COM EntryID. |
| index.html | GitHub Pages dashboard. CURRENTLY BROKEN — see Status below. |
| data/briefing.json | Generated by fetch_inbox.py Phase 3, pushed to GitHub on each run. |

## Current approach: Outlook COM

- Library: pywin32 (win32com.client)
- No OAuth. No tokens. Outlook handles auth.
- Pulls last 7 days of inbox, sent, calendar
- EntryID captured per email, injected into briefing by subject match
- Briefing pushed to GitHub after each run via Contents API

## openmail:// protocol handler

- Registered at HKCU:\Software\Classes\openmail in Windows registry
- Points to: "C:\Python314\python.exe" "C:\Users\admin\work-inbox\open_email.py" "%1"
- Chrome appends trailing / to protocol URLs — open_email.py strips it with .rstrip("/")
- Confirmed working end-to-end: browser click → Chrome prompt → Outlook opens exact email
